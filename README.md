# Подготовка датасета алгебраических утверждений из PDF

## Описание проекта

Программа предназначена для подготовки структурированного датасета по теме «Высказывания об алгебраических структурах». В качестве исходных данных используются PDF-файлы математических публикаций на русском языке.

Программа извлекает текст из PDF, выполняет его очистку, находит фрагменты с упоминаниями групп, колец, полей, модулей, идеалов, гомоморфизмов и изоморфизмов, после чего формирует записи датасета.

Для каждой найденной записи создаются:

* исходная формулировка на русском языке;
* предварительная англоязычная версия;
* шаблон формальной записи;
* заготовка кода Lean 4 + Mathlib;
* ключевые слова;
* категория утверждения;
* источник.

## Возможности программы

Программа реализует:

* обработку нескольких PDF-файлов;
* очистку текста от артефактов PDF;
* удаление служебных частей статьи и списка литературы;
* построение глоссария математических терминов;
* подсчёт встречаемости терминов и обозначений;
* извлечение предложений с математическими утверждениями;
* добавление соседнего контекста к найденной формулировке;
* удаление повторяющихся записей;
* классификацию утверждений по типам;
* создание JSON-, CSV- и XLSX-представлений датасета;
* создание файла для ручной аннотации;
* расчёт Cohen’s kappa после заполнения аннотаций.

## Категории утверждений

Программа использует следующие категории:

* `ISOMORPH` — утверждения об изоморфизмах;
* `SUBSTRUCTURE` — утверждения о подгруппах, идеалах, подмодулях и других подструктурах;
* `ACTION` — утверждения о действиях, орбитах и стабилизаторах;
* `CLASSIFICATION` — утверждения о классификации объектов;
* `PROPERTY` — утверждения о свойствах алгебраических структур.

## Структура проекта

```text
project/
├── main.py
├── pdfs/
│   ├── article1.pdf
│   ├── article2.pdf
│   └── ...
└── output/
```

`main.py` — основной файл программы.

`pdfs/` — каталог с исходными PDF-публикациями.

`output/` — каталог, в который программа сохраняет результаты обработки.

## Требования

Для запуска требуется Python 3.10 или новее.

Используемые библиотеки:

```text
pymupdf
pandas
openpyxl
```

## Установка зависимостей

В терминале необходимо выполнить команду:

```bash
pip install pymupdf pandas openpyxl
```

При использовании PowerShell в Windows можно выполнить:

```powershell
python -m pip install pymupdf pandas openpyxl
```

## Запуск программы

Перед запуском необходимо поместить исходные PDF-файлы в папку `pdfs`.

Запуск из корневого каталога проекта:

```powershell
python main.py
```

При необходимости пути можно указать явно:

```powershell
python main.py --pdf_dir pdfs --output_dir output
```

Программа обрабатывает все PDF-файлы, обнаруженные в указанной папке, включая вложенные каталоги.

## Выходные файлы

После выполнения программы в папке `output` формируются следующие файлы:

```text
output/
├── txt_clean/
│   ├── article1.txt
│   ├── article2.txt
│   └── ...
├── glossary.json
├── notation_table.csv
├── article_stats.csv
├── dataset.json
├── dataset_50.csv
├── Dataset.xlsx
├── annotation_pairs.csv
├── annotation_agreement.json
└── report.json
```

### `txt_clean/*.txt`

Очищенные текстовые версии исходных PDF-публикаций. Используются для проверки качества извлечения и дальнейшего формирования датасета.

### `glossary.json`

Глоссарий алгебраических объектов с определениями, примерами и найденными в публикациях фрагментами.

### `notation_table.csv`

Таблица встречаемых обозначений математических объектов и количества их упоминаний.

### `article_stats.csv`

Статистика употребления ключевых терминов в каждой обработанной статье.

### `dataset.json`

Основное структурированное представление итогового датасета. Количество записей ограничено значением, заданным в программе, и зависит от числа найденных подходящих утверждений.

### `dataset_50.csv`

Табличное представление первых 50 записей датасета.

### `Dataset.xlsx`

Excel-представление первых 30 записей датасета с настроенной шириной столбцов и переносом текста.

### `annotation_pairs.csv`

Файл для ручной разметки найденных утверждений двумя аннотаторами. Содержит столбцы:

```text
id
statement
predicted_type
annotator_1
annotator_2
```

### `annotation_agreement.json`

Результат вычисления коэффициента Cohen’s kappa после заполнения полей ручной разметки.

### `report.json`

Сводная информация о количестве обработанных файлов и сформированных записей.

## Поля датасета

Каждая строка итогового датасета содержит следующие поля:

| Поле                                            | Описание                                                 |
| ----------------------------------------------- | -------------------------------------------------------- |
| Формулировка на естественном языке (русский)    | Извлечённый фрагмент математического текста с контекстом |
| Формулировка на естественном языке (английский) | Предварительная англоязычная версия                      |
| Запись на формальном языке                      | Шаблон формальной математической записи                  |
| Код Lean 4 + Mathlib                            | Заготовка теоремы для формальной проверки                |
| Ключевые слова (русский / английский)           | Найденные тематические термины                           |
| Тип утверждения                                 | Назначенная категория утверждения                        |
| Источник                                        | Имя файла, из которого извлечена запись                  |

## Ручная аннотация и Cohen’s kappa

После первого запуска необходимо открыть файл:

```text
output/annotation_pairs.csv
```

Два аннотатора независимо заполняют столбцы:

```text
annotator_1
annotator_2
```

Допустимые категории:

```text
ISOMORPH
SUBSTRUCTURE
ACTION
CLASSIFICATION
PROPERTY
```

После заполнения файла программу необходимо запустить повторно. Результат расчёта согласия аннотаторов будет сохранён в:

```text
output/annotation_agreement.json
```

Целевое значение коэффициента согласия для работы — не ниже `0.70`.

## Ограничения реализации

Поле с англоязычной формулировкой создаётся с помощью словарной замены основных терминов и не является полноценным автоматическим переводом.

Формальные записи и код Lean 4 + Mathlib генерируются как тематические шаблоны. Они предназначены для дальнейшей ручной проверки и не гарантируют точного соответствия каждой исходной формулировке.

Классификация выполняется по регулярным выражениям и ключевым словам. Для получения качественного итогового датасета требуется ручная аннотация и проверка автоматически сформированных записей.
